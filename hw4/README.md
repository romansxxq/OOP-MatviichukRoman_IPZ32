# Домашня робоота №4
## Тема: Factory Method vs Abstract Factory

## Опис

У цьому проєкті розглядаються два породжувальні патерни проєктування: **Factory Method** та **Abstract Factory**.  
Вони використовуються для створення об’єктів і допомагають зробити код гнучким, зрозумілим та зручним для розширення.

---

## Factory Method

### Що це таке

Factory Method — це патерн, у якому створення об’єкта відбувається через спеціальний метод.  
Підкласи самі визначають, який саме об’єкт створювати.

### Коли використовується

- коли потрібно створювати один тип об’єкта з різними реалізаціями  
- коли наперед невідомо, який клас буде потрібен  
- коли потрібно легко додавати нові типи  

### Переваги

- простий у використанні  
- легко розширюється  
- зменшує залежність від конкретних класів  

### Недоліки

- з’являється багато підкласів  
- ускладнюється структура програми  

---

## Abstract Factory

### Що це таке

Abstract Factory — це патерн, який створює не один об’єкт, а ціле сімейство пов’язаних об’єктів.

### Коли використовується

- коли потрібно працювати з групами сумісних об’єктів  
- коли система має різні платформи або стилі  
- коли важлива узгодженість компонентів  

### Переваги

- гарантує сумісність об’єктів  
- легко змінювати сімейства  
- приховує реалізацію від клієнта  

### Недоліки

- складніший у реалізації  
- багато коду  
- важко додавати нові типи продуктів  

---

## Порівняння

| Критерій | Factory Method | Abstract Factory |
|----------|---------------|------------------|
| Тип створення | Один об’єкт | Група об’єктів |
| Складність | Низька | Вища |
| Гнучкість | Висока | Висока |
| Використання | Прості задачі | Складні системи |

---

## Коли який патерн обирати

### Factory Method

Використовується, якщо:

- потрібно створювати один тип об’єктів  
- проєкт невеликий  
- не потрібні групи пов’язаних класів  

Приклади: логер, парсер, валідатор.

### Abstract Factory

Використовується, якщо:

- є кілька пов’язаних об’єктів  
- проєкт підтримує різні платформи  
- потрібна сумісність компонентів  

Приклади: UI для різних ОС, теми, драйвери.

---

## Висновок

Factory Method — простий і зручний патерн для більшості задач.  
Abstract Factory — потужніший, але складніший і підходить для великих систем.

Вибір патерну залежить від складності проєкту та вимог до архітектури.
