# Домашнє завдання (Лекція №19)
## Тема: Composite і Decorator: концепції та компроміси.

---
### Composite

Патерн Composite дозволяє об’єднувати об’єкти у деревоподібні структури, де кожен елемент може бути як простим (листом), так і складеним (гілкою). Завдяки цьому, ієрархічні структури можна обробляти як єдину колекцію: клієнтський код працює з усіма елементами однаково, не розрізняючи листи та гілки.

**Реальні сценарії застосування:**
1. **Файлова система:** папки містять файли та інші папки, і всі вони мають спільний інтерфейс для операцій (наприклад, “відкрити”, “видалити”).
2. **Графічні інтерфейси:** складні компоненти (панелі, контейнери) містять прості елементи (кнопки, поля), і всі вони обробляються як єдина структура.
3. **Організаційна структура компанії:** менеджери можуть мати підлеглих, які теж можуть бути менеджерами або працівниками, і всі мають спільні методи (наприклад, “отримати зарплату”).

**Переваги:**
- Спрощує роботу з ієрархіями: клієнтський код не залежить від типу елементів.
- Легко розширювати структуру: додавати нові типи компонентів.
- Забезпечує гнучкість та масштабованість.

**Недоліки:**
- Може ускладнити дизайн через надмірну абстракцію.
- Важко контролювати поведінку окремих компонентів, якщо потрібно специфічне оброблення.

Composite — ефективний патерн для роботи з деревоподібними структурами, коли важливо обробляти їх як єдину колекцію.
