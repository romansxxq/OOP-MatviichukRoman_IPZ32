# Лабораторна робота No31. Варіант №8
## Тема: Тестування з Moq (мокінг залежностей).
## Мета: Навчитися створювати mock-об’єкти за допомогою Moq для тестування класів з залежностями, використовувати Setup та Verify.

## Завдання:
Для тестування програми, реалізовано тести для класу `ReportService` (див. [lab31v8/Program.cs](../lab31v8/Program.cs)).

### 1) Dependency Injection
- Клас `ReportService` використовує dependency injection через конструктор для підключення залежностей `IDataSource` та `IPdfGenerator`.

### 2) Тести з Moq
- Створено мінімум 8 тестів з використанням Moq.
- Використано `Setup` для налаштування mock-об'єктів.
- Використано `Verify` для перевірки викликів методів.

#### Перевірено:
- Виклик `GetData()` та `GeneratePdf()` при створенні звіту.
- Коректність даних, що передаються у `GeneratePdf()`.
- Поведінку при порожніх або null-даних.
- Відсутність зайвих викликів.
- Повторний виклик `CreateReport`.
- Обробку винятків при помилках у джерелі даних.

### 3) Типи тестів
- `[Fact]` — для одиночних сценаріїв.
- Moq — для перевірки взаємодії між залежностями.

[Глянути тести](/lab31v8.Tests/UnitTest1.cs)

## Результат
- Всі тести виконуються успішно.
- Перевірено взаємодію між класами через mock-об'єкти.

## Висновок
Під час виконання лабораторної роботи отримано практичні навички:
- написання юніт-тестів у xUnit;
- використання Moq для створення mock-об'єктів;
- перевірки викликів методів через `Verify`;
- тестування винятків;
- перевірки граничних і комплексних сценаріїв.
