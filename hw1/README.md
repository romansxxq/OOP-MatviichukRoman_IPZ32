# Домашня робота №1

### God Object – це анти-патерн у програмуванні, коли один клас намагається робити занадто багато речей. Такий клас зазвичай містить безліч методів та властивостей, відповідає за різні аспекти програми і стає важким для підтримки. Головна проблема God Object – він порушує принцип єдиної відповідальності (SRP), оскільки зміни в одній частині класу можуть несподівано впливати на інші.

Для прикладу, ось такий клас:
```csharp
class OrderManager
{
    public void CreateOrder() { /* логіка створення */ }
    public void CalculateDiscount() { /* логіка знижок */ }
    public void PrintInvoice() { /* друк рахунку */ }
}
```

Цей клас порушує SRP, бо він одночасно відповідає за управління замовленням, обрахунок знижок та друк рахунку. Через це будь-яка зміна у способі друку або знижок вимагатиме змін у самому OrderManager, що збільшує ризик помилок.

## Щоб дотриматися SRP, можна розбити клас на декілька:
```csharp
class OrderService
{
    public void CreateOrder() { /* логіка створення */ }
}

class DiscountCalculator
{
    public void CalculateDiscount() { /* логіка знижок */ }
}

class InvoicePrinter
{
    public void PrintInvoice() { /* друк рахунку */ }
}
```

Тепер кожен клас має одну відповідальність: `OrderService` працює з замовленнями, `DiscountCalculator` - з обчисленням знижок, `InvoicePrinter` - з друком рахунків. Це робить код більш зрозумілим, легким для підтримки та тестування.

## Висновок:
God Object створює складність і порушує SRP. Розділення класів за принципом єдиної відповідальності дозволяє писати чистий та підтримуваний код, зменшує помилки та полегшує розробку.
